Lesson 5: Semantic HTML

Introduction

Semantic HTML uses meaningful tags that describe the content's purpose, not just its appearance. This makes your website more accessible to screen readers, better for SEO, and easier for other developers to understand.

Key Semantic Elements

📄 Article (<article>)

Self-contained content like blog posts, news articles, or comments:

<article>
  <h2>Blog Post Title</h2>
  <p>Content of the post...</p>
</article>

📋 Section (<section>)

Groups related content with a clear theme:

<section>
  <h2>About Us</h2>
  <p>Company information...</p>
</section>

📌 Aside (<aside>)

Content related to main content but separate (sidebar, ads):

<aside>
  <h3>Related Links</h3>
  <ul>
    <li><a href="#">Link 1</a></li>
  </ul>
</aside>

🎯 Main (<main>)

The primary content area (only one per page):

<main>
  <h1>Page Title</h1>
  <p>Main page content...</p>
</main>

Complete Structure Example

<header>Site Header</header>
<main>
  <section>
    <article>Blog post content</article>
  </section>
  <aside>Sidebar content</aside>
</main>
<footer>Site Footer</footer>

Understanding Semantic Structure

Semantic HTML creates a logical hierarchy that both browsers and assistive technologies can understand:

🏗️ Document Structure
Clear hierarchy with meaningful containers
♿ Accessibility
Screen readers understand content purpose
🔍 SEO Benefits
Search engines better understand your content
👥 Team Collaboration
Other developers quickly understand structure

✅ Semantic Structure:

<main>
  <section>
    <h2>Blog Posts</h2>
    <article>
      <h3>Post Title</h3>
      <p>Post content...</p>
    </article>
  </section>
</main>

❌ Non-Semantic Structure:

<div>
  <div>
    <h2>Blog Posts</h2>
    <div>
      <h3>Post Title</h3>
      <p>Post content...</p>
    </div>
  </div>
</div>

⚠️ Generic divs provide no semantic meaning!

Your Task

Create a semantic blog page structure. Your page should include:

  • A <header> element with site title
  • A <main> element containing the primary content
  • A <section> inside main for grouping content
  • An <article> inside the section with a blog post
  • A <footer> element at the bottom

Validation requirements: Must contain <header>, <main>, <section>, <article>, and <footer>.
