Lesson 9: CSS Positioning

Introduction

CSS positioning allows you to precisely control where elements appear on the page. Understanding the different positioning methods gives you powerful control over your layout and enables advanced design techniques.

The Four Position Types

static
Normal document flow
relative
Offset from normal position
absolute
Positioned relative to nearest positioned parent
fixed
Fixed to viewport

Key Position Properties

üìç position: static

Default positioning - follows normal document flow

position: static;

üîÑ position: relative

Positioned relative to its normal position

position: relative; top: 10px;

üéØ position: absolute

Positioned relative to nearest positioned ancestor

position: absolute; top: 0; right: 0;

üìå position: fixed

Fixed position relative to viewport

position: fixed; bottom: 0;

‚úÖ Good Positioning Practice:

/* Sticky footer using fixed positioning */
.footer {
  position: fixed;
  bottom: 0;
  left: 0;
  width: 100%;
  background: #333;
  color: white;
  padding: 15px;
  z-index: 100;
}

/* Content margin to prevent footer overlap */
.main-content {
  margin-bottom: 80px;
}

‚ùå Poor Positioning Practice:

/* Using absolute without positioned parent */
.element {
  position: absolute;
  top: 50%;
  left: 50%;
  /* This will position relative to viewport, not parent */
}

/* Forgetting z-index for overlapping elements */
.overlay {
  position: fixed;
  /* No z-index - might appear behind other elements */
}
